💻 1. Executive Summary

As the UI/UX Lead for MineShield, I was responsible for the end-to-end visual and interactive architecture of the application—translating complex safety requirements into production-ready React Native components. My work directly supports core functional features for three specialized roles: Workers, Supervisors, and Visitors.

  • Key Deliverables: Fully realized Figma Design System, custom Cross-Platform Component Library, Global Core Theme Engine, and Technical System Documentation.

🎨 2. Figma Design System & Iteration

2.1 Environmental Design Factors

Designing for a harsh mining environment meant optimizing for poor lighting, gloved hands, and heavy display glare.

  • Touch Targets: Expanded all critical interactive targets to a minimum of 64 × 64 dp to ensure seamless accessibility for workers wearing heavy protective gear.
  • The Contrast Pivot: Early field tests revealed that a dark interface caused severe glare under direct outdoor light. I rapidly refactored the application's base canvas to a high-visibility light theme (#F3F6FC) to maximize readability.
  • Safety Fail-Safes: Redesigned the SOS trigger with a 3-second press-and-hold gesture and adjusted the automated fall-detection timeout modal to 10 seconds complete with a highly visible countdown progress bar.

⚙️ 3. Core Architecture & Code Implementation

All production components are written in React Native (Expo) and hook directly into a central context engine. To protect safety-critical UX, semantic colors (Critical Red, Warning Amber, Safe Green) remain completely fixed across all visual overrides.

3.1 The Dynamic Theme Engine

src/contexts/ThemeContext.js manages global spacing tokens, border radii, and a four-tier theme system saved persistently via AsyncStorage.

3.2 Key Reusable UI Components

  • Button.js: Features strict WCAG-compliant touch layouts (≥ 48 × 48 dp), an integrated async loading layout, and adaptive state variants (primary, secondary, danger, warning, outline).
  • Card.js: Built as an explicit semantic layout container. While default states adopt standard theme surfaces, explicit safety alerts strictly overwrite containers with uncompromised absolute fills (danger → #D46B5A).
  • Header.js: Formats structured application navigation, supplying deep layout slot handling for contextual back-navigation, views, and action indicators.

🤝 5. Engineering Leadership & Technical Challenges

Overcoming Fragmented Styling Styles

The Problem: Developers on backend and logic tracks initially bypassed UI layouts by hardcoding miscellaneous styling rules directly into core functional loops.

The Solution: I established a strict project gatekeeping workflow: No component pull requests were merged into production without an approved, mapped Figma design token. I then authored extensive markdown specifications (DESIGN_SYSTEM.md and `UI_COMPONENTS.md`), serving as a central dictionary that smoothed out team integration.
